## Directory Purposes

**`src/pages/`:**
- Purpose: URL structure and route-level logic only; prefer thin pages that compose layouts and components.
- Contains: `.astro` pages, `src/pages/api/**/*.ts` API routes, `rss.xml.js`.
- Key files: `src/pages/index.astro`, `src/pages/blog/[...slug].astro`, `src/pages/api/contact.ts`

**`src/layouts/`:**
- Purpose: Shared page wrappers; one primary layout for the public site.
- Contains: Astro-only layout components.
- Key files: `src/layouts/BaseLayout.astro`

**`src/components/`:**
- Purpose: UI split between Astro (static/markup) and React (interactivity). `ui/` holds small composable controls aligned with shadcn-style patterns.
- Contains: Feature sections (`Hero.client.tsx`, `Projects.client.tsx`), `Navigation.tsx`, MDX helpers under `mdx/`.
- Key files: `src/components/Navigation.tsx`, `src/components/CommandPalette.tsx`, `src/components/ui/button.tsx`

**`src/content/`:**
- Purpose: Authoring location for collections defined in `src/content.config.ts`.
- Contains: Markdown/MDX with frontmatter matching Zod schemas.
- Key files: per-post files such as `src/content/blog/*.md`, `src/content/journal/*.md`

**`src/lib/`:**
- Purpose: Non-UI logic — site config, formatting, API helpers, feature datasets, graph builder, optional SQL helpers.
- Contains: `config.ts`, `utils.ts`, `api-utils.ts`, `graph.ts`, `*-data.ts`, `db/index.ts`, `db/schema.sql`
- Key files: `src/lib/config.ts`, `src/lib/graph.ts`, `src/lib/api-utils.ts`

**`src/types/`:**
- Purpose: TypeScript interfaces shared across components and config.
- Contains: `index.ts` barrel and feature-specific types (e.g. `roadmap-voting.ts`).
- Key files: `src/types/index.ts`

**`src/hooks/`:**
- Purpose: React hooks reused by multiple `.client.tsx` components.
- Contains: `useDebounce.ts`, `useScrollSpy.ts`, etc.

**`scripts/`:**
- Purpose: Offline automation (journal summaries, content indexing for search).
- Contains: Bun TypeScript scripts; invoked via `package.json` scripts.

**`public/`:**
- Purpose: Assets referenced by URL path (e.g. `/favicon.svg`, CV PDFs).
- Generated: No
- Committed: Yes (intentional static assets)

**`data/`:**
- Purpose: Local database/index files (e.g. LanceDB) used by indexing or runtime search depending on deployment.
- Generated: Often yes (regenerated by scripts)
- Committed: Typically gitignored or partially untracked — treat as environment output, not source of truth for prose content.

## Key File Locations

**Entry Points:**
- `astro.config.mjs`: Build and integration configuration.
- `src/pages/index.astro`: Landing page.
- `src/pages/api/*.ts` and nested folders: Server handlers.

**Configuration:**
- `tsconfig.json`: `@/*` → `src/*`
- `src/content.config.ts`: Content collection definitions
- `src/lib/config.ts`: Site copy, SEO, navigation, portfolio data

**Core Logic:**
- `src/lib/graph.ts`: Knowledge graph data from collections
- `src/lib/api-utils.ts`: HTTP retry/compression helpers for API routes
- `src/lib/db/schema.sql` + `README.md`: reference DDL for a possible future D1 deployment (no runtime TS client in-repo)

**Testing:**
- Example unit test: `src/lib/utils.test.ts` — co-located with `src/lib/utils.ts`

## Naming Conventions

**Files:**
- **Routes:** `index.astro` for section roots; `[...slug].astro` for catch-all dynamic segments; API files named by HTTP concern (`contact.ts`, `webhook.ts`).
- **React islands:** `FeatureName.client.tsx` signals a client-hydrated island (paired with `client:load` / `client:visible` / `client:idle` in parent `.astro`).
- **Shared components:** `PascalCase.tsx` or `PascalCase.astro` (e.g. `Navigation.tsx`, `BaseLayout.astro`).
- **Content:** kebab-case or date-prefixed filenames under `src/content/**`; IDs derive from paths (files starting with `_` are excluded by glob patterns in `content.config.ts`).

**Directories:**
- Lowercase with hyphens where multi-word at route level (`bug-bounty` under API); `ui/`, `mdx/`, `seo/` as feature buckets inside `components/`.

## Where to Add New Code

**New Feature:**
- Primary code: New `.astro` under `src/pages/` (or nested folder) and components under `src/components/`; shared logic in `src/lib/`.
- Types: `src/types/` or colocated `types.ts` if scope is tiny and private to one feature.

**New Component/Module:**
- Implementation: Feature-specific → `src/components/`; design-system primitive → `src/components/ui/`; layout wrapper → `src/layouts/`.

**New Content Type:**
- Add a `defineCollection` in `src/content.config.ts`, create folder under `src/content/<name>/`, add listing and `[...slug].astro` (or single page) under `src/pages/<name>/`.

**Utilities:**
- Shared helpers: `src/lib/utils.ts` or new module `src/lib/<topic>.ts` if substantial.

**API Endpoint:**
- Add `src/pages/api/<segment>/index.ts` or `src/pages/api/<name>.ts`; export `GET`/`POST` as `APIRoute`; set `export const prerender = false` for runtime execution.

**Tests:**
- Co-locate `*.test.ts` next to the module under test (see `src/lib/utils.test.ts`).

## Special Directories

**`.astro/` (generated):**
- Purpose: Type generation for content collections and Astro internals.
- Generated: Yes (by Astro)
- Committed: Project-dependent — often gitignored; do not hand-edit.

**`src/pages/*.disabled`:**
- Purpose: Routes temporarily removed from the build by renaming; restore by renaming to `.astro` when re-enabling.
